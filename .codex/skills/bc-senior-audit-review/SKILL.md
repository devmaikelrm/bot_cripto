# bc-senior-audit-review

1. Review repo by severity and produce critical findings first.
2. Evaluate:
   - architecture cohesion
   - config consistency
   - secrets and security controls
   - edge cases and runtime failures
   - test gaps
   - performance bottlenecks
3. Output report with:
   - high/medium/low issues
   - concrete remediation actions
   - prioritization order
   - safe diff suggestions
4. Avoid cosmetic-only review output.
